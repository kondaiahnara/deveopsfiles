Your outlined steps for installing Apache Tomcat on Ubuntu are generally correct, but there are a few adjustments and clarifications to ensure a smooth installation:

1.System Update:

It's good practice to update your system packages before starting the installation:

sudo apt update

sudo apt upgrade -y

2.Install Java Development Kit (JDK):

Apache Tomcat requires Java to run. It's recommended to install OpenJDK 17 or later:

sudo apt install openjdk-17-jdk -y

Verify the installation:
java -version

3.Create Tomcat User and Group:

For security reasons, it's advisable to run Tomcat under a dedicated user and group:

sudo groupadd tomcat

sudo useradd -s /bin/false -g tomcat -d /opt/tomcat tomcat

Download and Install Apache Tomcat:

Navigate to the /tmp directory and download the desired version of Tomcat. At the time of writing, Tomcat 10.1.14 is a stable release:

cd /tmp

sudo wget https://archive.apache.org/dist/tomcat/tomcat-10/v10.1.14/bin/apache-tomcat-10.1.14.tar.gz

Extract the downloaded archive to the /opt/tomcat directory:

sudo mkdir -p /opt/tomcat
sudo tar -xzvf apache-tomcat-10.1.14.tar.gz -C /opt/tomcat --strip-components=1

4.Set Permissions:

Assign ownership of the Tomcat directory to the tomcat user and group:


sudo chown -R tomcat:tomcat /opt/tomcat

5.Configure Tomcat Users for Web Interface Access:

To access the Tomcat Manager and Host Manager web interfaces, define users and roles:

Edit the tomcat-users.xml file:

sudo nano /opt/tomcat/conf/tomcat-users.xml

Add the following lines before the closing </tomcat-users> tag:

xml

<role rolename="manager-gui"/>
<role rolename="admin-gui"/>
<user username="manager" password="Admin@123" roles="manager-gui"/>
<user username="admin" password="Admin@123" roles="manager-gui,admin-gui"/>
Replace secure-password with strong passwords of your choice.

6.Allow Remote Access to Manager and Host Manager:

By default, access to the Manager and Host Manager apps is restricted to localhost. To allow remote access:

Edit the context.xml file for each application:

sudo nano /opt/tomcat/webapps/manager/META-INF/context.xml

sudo nano /opt/tomcat/webapps/host-manager/META-INF/context.xml

Locate and remove or comment out the following line:


<Valve className="org.apache.catalina.valves.RemoteAddrValve"
       allow="127\.\d+\.\d+\.\d+|::1|0:0:0:0:0:0:0:1" />
Create a Systemd Service File for Tomcat:

7.To manage Tomcat as a systemd service:

Create and edit the service file:

sudo nano /etc/systemd/system/tomcat.service

Add the following content:

ini
Copy

[Unit]
Description=Apache Tomcat 10 Web Application Server
After=network.target

[Service]
Type=forking
User=tomcat
Group=tomcat
Environment="JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64"
Environment="CATALINA_HOME=/opt/tomcat"
Environment="CATALINA_BASE=/opt/tomcat"
Environment="CATALINA_PID=/opt/tomcat/temp/tomcat.p
Environment="CATALINA_OPTS=-Xms512M -Xmx1024M -server -XX:+UseParallelGC"
ExecStart=/opt/tomcat/bin/startup.sh
ExecStop=/opt/tomcat/bin/shutdown.sh

[Install]
WantedBy=multi-user.target

8.Ensure that JAVA_HOME points to your installed Java version.

Reload Systemd, Start Tomcat, and Enable at Boot:

Reload systemd to recognize the new service, start Tomcat, and enable it to start on boot:

sudo systemctl daemon-reload
sudo systemctl start tomcat
sudo systemctl enable tomcat
